# 医疗仓PCB功能测试方案

> 本方案在联调业务固件之前执行，目的是验证PCB硬件本身的功能。
> 分4个阶段递进，**每阶段通过才进入下一阶段**。
> 所需工具：可调电源 + 万用表 + J-Link + 逻辑分析仪 + USB转TTL

---

## 阶段一：裸板上电检查（不烧固件，不接传感器）

### 目的
排除供电电路、焊接、短路等制造缺陷。

### 步骤

#### 1.1 目视检查
- ☐ PCB无明显焊接缺陷（桥连、虚焊、漏焊）
- ☐ MCU芯片方向正确（Pin1标记与PCB丝印一致）
- ☐ 8MHz晶振已焊接
- ☐ 32.768kHz晶振已焊接（如有）
- ☐ 所有电容/电阻/电感位置正确
- ☐ ULN2003A(U5)、ULN2001(U34)、ULN2001(U33)已焊接且方向正确
- ☐ 继电器已焊接
- ☐ 供电LDO/DC-DC已焊接

#### 1.2 短路检查（不上电）
用万用表二极管/通断档：
- ☐ 3.3V与GND之间不短路（阻值>1kΩ）
- ☐ 5V与GND之间不短路
- ☐ 12V与GND之间不短路（如有）
- ☐ 3.3V与5V之间不短路

#### 1.3 上电测量
使用**可调电源**供电（设限流300mA）：

| 测试点 | 预期电压 | 容差 | 实测 | 结果 |
|--------|---------|------|------|------|
| 输入电压（USB/DC） | 5.0V | ±0.5V | ___V | ☐ |
| LDO输出（VCC_3V3） | 3.3V | ±0.1V | ___V | ☐ |
| MCU VDD | 3.3V | ±0.1V | ___V | ☐ |
| MCU VDDA | 3.3V | ±0.1V | ___V | ☐ |

**电流检查**：
- ☐ 空载电流 < 50mA（MCU+LDO待机，无固件）
- ☐ 如果电流异常大（>200mA）→ 停止，检查短路

#### 1.4 晶振检查
用示波器探头轻触晶振引脚（OSC_IN/OSC_OUT）：
- ☐ 上电后晶振是否起振？（注意：MCU无固件时晶振可能不起振，这是正常的）
- ☐ 如果用最小固件后检查，应看到8MHz正弦波

### 阶段一通过标准
- 所有供电电压在容差内
- 空载电流正常
- 无短路

---

## 阶段二：MCU连通性测试（烧最小固件）

### 目的
验证MCU能烧录、运行、UART通信正常、时钟正确。

### 步骤

#### 2.1 J-Link/SWD连接
接线：J-Link → PCB SWD接口（SWDIO/SWCLK/GND/3V3）

```bash
# 检测MCU
JLink.exe -device STM32F103VE -if SWD -speed 1000 -autoconnect 1
```

- ☐ 显示 `Cortex-M3 r1p1, Little endian`
- ☐ VTref ≈ 3.3V
- ☐ 如果显示Big endian或连接失败 → SWDIO/SWCLK可能接反

#### 2.2 烧录诊断固件
使用 JFlash Lite 烧录 `main_diag.c` 编译产物：
- Device: STM32F103VE
- File: diag.bin（约3KB）
- Address: 0x08000000

- ☐ Program Device → Verify successful

#### 2.3 UART1输出检查
逻辑分析仪或USB转TTL接 PA9（UART1_TX），115200 8N1：

预期输出：
```
================================
[DIAG] HDICU Diagnostic Start
[DIAG] UART1: 115200 on PA9
================================
[DIAG] HSE/PLL test...
  HSE+PLL: OK (或 HSE: FAIL)
  HSEON=1 HSERDY=1 PLLON=1 PLLRDY=1
  SWS=2 (PLL)
  SYSCLK=72MHz PCLK1=36MHz
[DIAG] GPIO PE2/PE3/PE4...OK
[DIAG] ADC init...OK
[DIAG] TIM6 init...OK
[DIAG] All peripheral tests done
[DIAG] alive cnt=0
[DIAG] alive cnt=1
...
```

检查项：
- ☐ UART1输出可读（无framing error）
- ☐ HSE是否启动成功？（HSEON=1, HSERDY=1）
- ☐ PLL是否锁定？（PLLON=1, PLLRDY=1）
- ☐ SYSCLK是72MHz还是8MHz？
- ☐ GPIO/ADC/TIM6初始化是否OK？

### 阶段二通过标准
- MCU能烧录和运行
- UART1 115200输出正确
- 知道当前时钟状态（72MHz或8MHz）

## 阶段三：GPIO逐路测试

### 目的
确认9路继电器、3路PWM、3路LED、蜂鸣器、传感器UART的GPIO物理连通性。

### 步骤

#### 3.1 烧录PCB测试固件
使用 `main_pcb_test.c` 编译产物（约4KB）。

#### 3.2 继电器逐路测试
测试固件会逐路翻转9个继电器GPIO（高→低→高），每路停2秒。

用万用表或听继电器吸合声验证：

| 逻辑ID | 信号名 | 代码GPIO | 继电器动作? | 实测 |
|--------|--------|---------|-----------|------|
| 0 | PTC-IO | PD13(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 1 | JIARE-IO | PD14(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 2 | RED-IO | PD15(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 3 | ZIY-IO | PE7(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 4 | O2-IO | PE8(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 5 | JIASHI-IO | PE9(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 6 | FENGJI-IO | PE10(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 7 | YASUO-IO | PE11(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |
| 8 | WH-IO | PE12(占位) | ☐吸合 ☐不动 | → 实际GPIO=____ |

**如果继电器不动**：
1. 用万用表量MCU引脚→ULN输入→ULN输出→继电器线圈的通路
2. 对照原理图找到正确的MCU引脚
3. 记录到"实际GPIO"列
4. 测试结束后更新 `relay_driver.c`

#### 3.3 PWM输出测试
逻辑分析仪接PE2/PE3/PE4，测试固件会输出不同占空比PWM：

- ☐ PE2：看到100Hz PWM，占空比30%→60%→100%循环
- ☐ PE3：同上
- ☐ PE4：同上

#### 3.4 LED测试
- ☐ PD10 护理灯1 → 亮/灭
- ☐ PD11 护理灯2 → 亮/灭
- ☐ PD12 护理灯3 → 亮/灭

#### 3.5 蜂鸣器测试
- ☐ PB0 → 蜂鸣器响/停

#### 3.6 外循环/新风GPIO查找
- ☐ 对照原理图，找到控制内/外循环电磁阀的MCU GPIO
- ☐ 对照原理图，找到控制新风净化电磁阀的MCU GPIO
- ☐ 记录到HARDWARE_PIN_MATRIX.md

### 阶段三通过标准
- 所有9路继电器的真实GPIO已确认
- PWM波形可测量
- LED/蜂鸣器正常
- 外循环/新风GPIO已查找

## 阶段四：外设通路测试

### 目的
验证5路UART收发、ADC精度、继电器驱动能力。

### 步骤

#### 4.1 UART逐路测试
用USB转TTL逐路连接：

| UART | 引脚 | 设备 | 测试方法 | 结果 |
|------|------|------|---------|------|
| UART1 | PA9/PA10 | 屏幕板 | 连PC串口，看诊断输出 | ☐ |
| UART2 | PA2/PA3 | iPad | 连PC串口，发0x01看回包 | ☐ |
| UART3 | PB10/PB11 | CO2 | 连CO2传感器，看FF 17帧 | ☐ |
| UART4 | PC10/PC11 | O2 | 连O2传感器，看16 09帧 | ☐ |
| UART5 | PC12/PD2 | JFC103 | 连JFC103，发0x8A看88字节 | ☐ |

#### 4.2 ADC精度测试
用可调电源给ADC引脚施加已知电压：

| 通道 | 施加电压 | 预期ADC值 | 预期温度 | 实测 |
|------|---------|----------|---------|------|
| PA0 | 1.65V (中点) | ~2048 | ~25°C | ☐ |
| PA0 | 1.0V | ~1241 | ~46°C | ☐ |
| PA0 | 2.5V | ~3103 | ~5°C | ☐ |

#### 4.3 继电器驱动能力测试（低压负载）
⚠️ 先用12V低压灯泡或LED+电阻作为负载，不要直接接220V！

- ☐ 每路继电器能驱动12V/100mA负载
- ☐ 继电器吸合/释放声音正常
- ☐ ULN芯片不发烫

#### 4.4 Flash读写测试
- ☐ 读0x0807F000区域（应为0xFF或上次写入值）
- ☐ 写入测试数据→读回一致
- ☐ 掉电→重新上电→读回一致

### 阶段四通过标准
- 5路UART均能收发数据
- ADC精度满足0.1°C分辨率需求
- 继电器驱动正常
- Flash存储可靠

## 常见问题排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 上电无电压 | LDO未焊/方向反 | 量LDO输入输出 |
| 电流异常大 | 短路 | 逐区域排查 |
| J-Link连不上 | SWD接线错/MCU虚焊 | 检查SWDIO/SWCLK/GND |
| 晶振不起振 | 晶振虚焊/负载电容错 | 换晶振/检查电容值 |
| UART无输出 | GPIO复用错/波特率错 | 逻辑分析仪量波特率 |
| 继电器不吸合 | GPIO引脚错/ULN方向反 | 万用表追踪通路 |
| ADC值固定0或4095 | 引脚浮空/分压电阻缺失 | 量引脚电压 |
| Flash写入失败 | 地址错/页大小错 | 检查0x0807F000 |
