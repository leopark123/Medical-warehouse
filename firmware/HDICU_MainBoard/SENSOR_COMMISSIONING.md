# 传感器联调指南

## 逐路联调顺序

建议按以下顺序逐个接入传感器，每接一个确认正常后再接下一个：
1. NTC温度（最简单，ADC直连）
2. CO2传感器（UART3，自动上报）
3. O2传感器（UART4，自动上报）
4. JFC103体征（UART5，需发启动命令）

---

## 1. NTC温度 (4路ADC)

### 接线
| 通道 | MCU引脚 | 接NTC | 分压电阻 |
|------|---------|-------|---------|
| NTC1 | PA0 | NTC一端接PA0，另一端接GND | 10kΩ从PA0到3.3V |
| NTC2 | PA1 | 同上 | 同上 |
| NTC3 | PA4 | 同上 | 同上 |
| NTC4 | PA5 | 同上 | 同上 |

### 验证方法
- 观察debug输出中 `T=XX.X` 值
- 用温度计对比，误差应<2°C
- 手指捏NTC → 温度应上升
- NTC断开 → T=-99.9（无效标记）

### 公式
```
R_ntc = 10000 × ADC / (4095 - ADC)
T(K) = 1 / (1/298.15 + ln(R_ntc/10000)/3950)
T(°C) = T(K) - 273.15
```

### 排查
- ADC值始终0或4095 → 检查接线/分压电阻
- 温度偏差大 → 确认NTC B值是3950、标称10kΩ
- 只有部分通道工作 → 检查对应GPIO是否被其他功能占用

## 2. CO2传感器 MWD1006E (UART3)

### 接线
| CO2模块 | MCU |
|---------|-----|
| TX | PB11 (UART3_RX) |
| RX | PB10 (UART3_TX) — 通常不需要 |
| VCC | 5V |
| GND | GND |

### 协议
- 波特率: 9600, 8N1
- 工作模式: 自动上报，每4秒一帧
- 帧格式: `FF 17 04 00 [CO2_H] [CO2_L] 00 00 [CS]` (9字节)
- 校验: `CS = (~(sum bytes[1..7]) + 1) & 0xFF`
- CO2值: `(CO2_H << 8) | CO2_L`，单位ppm

### 验证方法
1. 用逻辑分析仪抓PB11，应每4秒看到一帧数据
2. 解码确认帧头FF 17
3. 观察debug输出中 `CO2=XXX` 值
4. 室内正常值: 400-1000 ppm
5. 对着传感器呼气 → CO2值应显著上升

### 排查
- 无数据 → 检查TX/RX是否交叉连接、供电是否5V
- CO2=0 → 检查校验计算、帧同步
- 超时失效(10s) → 检查UART3波特率9600

## 3. O2传感器 OCS-3RL-2.0 (UART4)

### 接线
| O2模块 | MCU |
|--------|-----|
| TX | PC11 (UART4_RX) |
| RX | PC10 (UART4_TX) — 通常不需要 |
| VCC | 5-12V |
| GND | GND |

### 协议
- 波特率: 9600, 8N1
- 工作模式: 自动上报，每秒2帧(500ms间隔)
- 帧格式: 12字节，帧头 `16 09 01/02`
- 校验: 所有12字节累加和低8位=0x00
- 数据(均需÷10):
  - Byte[3-4]: O2浓度 (÷10 = %)
  - Byte[5-6]: 湿度 (÷10 = %RH)
  - Byte[7-8]: 温度 (÷10 = °C)
  - Byte[9-10]: 气压 (÷10 = kPa)

### 验证方法
1. 逻辑分析仪抓PC11，应每500ms一帧
2. 确认帧头16 09
3. 观察debug输出中 `O2=XX H=XX` 值
4. 空气中O2≈20.9%
5. 超时失效(3s)

### 排查
- 无数据 → 检查供电(5-12V)、TX/RX交叉
- O2=0 → 检查帧头同步、mode字段(01或02)
- 数据偶尔错 → 检查校验和计算

## 4. JFC103体征模块 (UART5)

### 接线
| JFC103 | MCU |
|--------|-----|
| UTX (pin4) | PD2 (UART5_RX) |
| URX (pin3) | PC12 (UART5_TX) |
| VCC_IO (pin1) | 3.3V |
| GND (pin6) | GND |

> ⚠️ 原理图UART5标注为WiFi，但冻结基线已确定用于JFC103。

### 协议
- 波特率: 38400, 8N1
- 启动命令: 发送 `0x8A`
- 停止命令: 发送 `0x88`
- 数据帧: 88字节，帧头 `0xFF`
- 同步特性: 数据中不含其他0xFF
- 关键字段:
  - Byte[65]: 心率 (bpm)
  - Byte[66]: 血氧 (%)
  - Byte[67]: 微循环
- 上报周期: 1.28秒/帧
- 超时失效: 5s

### 验证方法
1. 固件启动后自动发送0x8A启动命令
2. 手指贴传感器透镜
3. 等3-5秒（预热）
4. 观察debug输出中 `HR=XX SpO2=XX` 值
5. 正常静息: HR=60-100, SpO2=95-100

### 排查
- HR=0 SpO2=0 → 确认0x8A已发送、手指接触良好
- 数据乱跳 → 改善手指与传感器接触
- 无帧收到 → 检查UART5 38400波特率、TX/RX交叉

### 不使用的字段
JFC103还输出血压(byte71-72)、呼吸率等，但当前项目冻结为"仅心率+血氧"。
iPad协议0x06响应中字节6-13固定填0。

## 传感器有效性与报警

所有传感器都有超时失效机制：

| 传感器 | 超时 | 报警影响 |
|--------|------|---------|
| CO2 | 10秒 | co2_valid=false → CO2报警不触发 |
| O2 | 3秒 | o2_valid=false → O2/湿度报警不触发，氧控冻结 |
| JFC103 | 5秒 | jfc103_valid=false → HR/SpO2报警不触发 |
| NTC | 无超时 | temperature_avg=-999时跳过 |

传感器离线时：
- 控制逻辑冻结当前状态（不基于陈旧数据做决策）
- 报警不产生虚假警报
- iPad 0x02响应中传感器值保持最后有效值或0
