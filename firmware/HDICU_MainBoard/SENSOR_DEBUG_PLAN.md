# 传感器联调方案（零基础版）

> 前提：正式固件已烧录，9路继电器/蜂鸣器/护理灯/GPIO全部通过PCB验证。
> 本方案逐个接入传感器，每接一个验证通过再接下一个。

---

## 工具准备

| 工具 | 用途 | 你有吗 |
|------|------|--------|
| 逻辑分析仪(Telesky 24MHz 8ch) | 抓UART数据帧验证 | ✅ |
| USB-TTL串口模块(CH343) | 看debug输出 | ✅ |
| 万用表 | 量电压/检查接线 | ✅ |
| J-Link + SWD线 | 烧录（如需换固件） | ✅ |
| 12V电源 | PCB供电 | ✅ |

---

## 重要限制

**HDICU_DEBUG固件的debug输出走UART4(CN16/PC10)。**
**O2传感器也用UART4(CN16/PC10)。**

所以：
- ✅ NTC联调时可以用debug固件 + USB-TTL接CN16看温度
- ✅ CO2联调时可以用debug固件 + USB-TTL接CN16看CO2值
- ❌ **O2联调时不能用debug固件** — debug文本会污染O2传感器通信
- ✅ JFC103联调时可以用debug固件 + USB-TTL接CN16看HR/SpO2

**O2联调方案：** 用逻辑分析仪直接抓UART4 TX/RX波形解码，不经过USB-TTL。

---

## 联调顺序

```
第1步：NTC温度（最简单，ADC，无通信协议）
   ↓
第2步：CO2传感器（UART3，自动上报，不冲突）
   ↓
第3步：JFC103体征（UART5，需启动命令）
   ↓
第4步：O2传感器（UART4，需要特殊方法验证）
```

---

## 第1步：NTC温度联调

### 需要什么
- NTC热敏电阻(10kΩ@25°C, B=3950) × 1~4个
- 10kΩ上拉电阻 × 1~4个（可能PCB上已有）

### PCB接口
| 接口 | MCU引脚 | 信号 |
|------|---------|------|
| NTC01(CN9) | PA0 | 顶部温度 |
| NTC02(CN11) | PA1 | 空调进风口温度 |
| NTC03 | PA4 | 温度3 |
| NTC04 | PA5 | 温度4 |

### 操作步骤

**准备：**
1. 断12V电源
2. 先确认PCB上NTC接口的分压电阻(10kΩ上拉到3.3V)是否已焊好
   - 万用表量PA0到3.3V之间电阻，应该是10kΩ
   - 如果没有，需要外接10kΩ电阻

**接线：**
3. NTC电阻一端接到NTC01接口的信号针
4. NTC电阻另一端接GND
5. 先只接1路(NTC01/PA0)测试

**烧debug固件：**
6. 断12V，只留J-Link
7. 烧录debug版本（带-DHDICU_DEBUG）
8. 拔J-Link
9. USB-TTL接CN16(TX→RX, GND→GND)
10. 接12V电源

**验证：**
11. 串口应显示类似：`[HDICU] up=5s T=25.3 H=0 O2=0 CO2=0 HR=0 SpO2=0`
12. T=XX.X就是NTC温度（x10格式除以10）
13. 用手捏NTC → T值应该上升
14. 松手 → T值应该回落
15. 拔掉NTC → T=-99.9（无效标记）
16. 室温范围：20-30°C正常

**通过标准：**
- [ ] T值在合理范围(20-30°C)
- [ ] 手捏后T上升
- [ ] 断开NTC后T=-99.9
- [ ] 如果接多路NTC，每路独立读数正常

## 第2步：CO2传感器联调

### 需要什么
- CO2传感器模块 MWD1006E
- 4根杜邦线

### PCB接口
CO2传感器UART数据走**CN5**(UART3, PB10/PB11, 9600)

### 接线
| CO2模块引脚 | 接到PCB |
|------------|---------|
| VCC | 5V（从CN5或其他5V接口取电） |
| GND | GND |
| TX | CN5的RX针脚（PB11方向） |
| RX | CN5的TX针脚（PB10方向）— 可以不接 |

> **注意TX/RX交叉：** 传感器的TX接MCU的RX，传感器的RX接MCU的TX。

### 操作步骤

**（如果还在跑NTC的debug固件，不用重新烧录，继续用）**

1. 断12V电源
2. 接CO2传感器到CN5
3. 接12V电源
4. USB-TTL仍接CN16看debug输出

**验证：**
5. 串口应显示：`[HDICU] up=10s T=25.3 H=0 O2=0 CO2=450 HR=0 SpO2=0`
6. CO2=XXX就是ppm值
7. 室内正常值：400-1000 ppm
8. 对着传感器呼气 → CO2值应显著上升（可能到2000-5000）
9. 等4秒看CO2是否刷新（MWD1006E每4秒上报一次）

**如果CO2一直=0：**
- 用逻辑分析仪CH0接CN5 RX(PB11)，看有没有数据帧
- 设9600 8N1解码，应看到 `FF 17 04 00 ...` 帧头
- 如果没有帧 → 检查传感器供电和TX/RX接线
- 如果有帧但固件不识别 → 检查帧格式/校验和

**通过标准：**
- [ ] CO2值在400-1000 ppm范围
- [ ] 呼气后CO2上升
- [ ] 每4秒刷新一次

## 第3步：JFC103体征模块联调

### 需要什么
- JFC103体征模块
- 杜邦线

### PCB接口
JFC103走UART5(PC12/PD2, 38400)

### 接线
| JFC103引脚 | 接到PCB |
|-----------|---------|
| VCC_IO (pin1) | 3.3V |
| URX (pin3) | PC12方向（UART5_TX） |
| UTX (pin4) | PD2方向（UART5_RX） |
| GND (pin6) | GND |

> 确认JFC103供电是3.3V不是5V！

### 操作步骤

1. 断12V电源
2. 接JFC103到UART5接口
3. 接12V电源
4. USB-TTL仍接CN16看debug输出

**验证：**
5. 固件启动后会自动发送0x8A启动命令
6. **手指贴在JFC103的传感器透镜上**（红光探头）
7. 等3-5秒预热
8. 串口应显示：`[HDICU] up=20s T=25.3 H=0 O2=0 CO2=450 HR=72 SpO2=98`
9. HR=心率(bpm)，SpO2=血氧(%)
10. 正常静息：HR=60-100, SpO2=95-100

**如果HR=0 SpO2=0：**
- 确认手指贴紧传感器（需要看到红光穿透手指）
- 用逻辑分析仪接PD2(UART5_RX)，38400解码，应看到88字节帧（帧头0xFF）
- 如果无帧 → 检查0x8A启动命令是否发出（逻辑分析仪接PC12看TX）

**通过标准：**
- [ ] HR=60-100, SpO2=95-100
- [ ] 拿开手指后HR/SpO2归零

## 第4步：O2传感器联调

### ⚠️ 特殊：不能用debug固件

**O2传感器和debug输出都用UART4(CN16)，会冲突。**

方案：用正式固件（无debug输出）+ 逻辑分析仪直接看UART4数据。

### 需要什么
- O2传感器模块 OCS-3RL-2.0
- 逻辑分析仪

### PCB接口
O2传感器UART数据走**CN16**(UART4, PC10/PC11, 9600)

### 操作步骤

**烧回正式固件（非debug）：**
1. 断12V，拔USB-TTL
2. 只留J-Link，烧录 build/firmware.bin（之前编译好的正式固件）
3. 拔J-Link

**接线：**
4. O2传感器接CN16（TX→RX, RX→TX, VCC→5-12V, GND→GND）
5. 逻辑分析仪CH0接CN16 RX方向（PC11引脚侧），看O2传感器发来的数据
6. 逻辑分析仪GND接PCB GND
7. 接12V电源

**验证（用逻辑分析仪）：**
8. 逻辑分析仪设9600 8N1解码
9. 应每500ms看到一帧数据
10. 帧头：`16 09 01` 或 `16 09 02`
11. 帧长12字节
12. Byte[3-4]: O2浓度(÷10)，空气中≈209(=20.9%)
13. Byte[5-6]: 湿度(÷10)
14. Byte[7-8]: 温度(÷10)

**如何确认固件正确处理了O2数据：**

方法A：看继电器行为
- 如果O2传感器接入后o2_valid变true，湿控和氧控会开始工作
- 设一个湿度目标值，看JIASHI继电器(U18)是否响应

方法B：用iPad协议读取
- USB-TTL接CN4(iPad UART2, PA2/PA3)
- 发送iPad查询命令 `AA 01 00 01 55`（0x01读参数）
- 收回34字节应答，其中包含O2/湿度值

方法C：拔O2传感器后看变化
- 拔掉O2传感器 → 3秒后o2_valid变false → 氧控/湿控冻结

**通过标准：**
- [ ] 逻辑分析仪看到16 09帧头的12字节帧
- [ ] O2浓度≈20.9%
- [ ] 帧校验和正确（12字节累加低8位=0x00）

---

## 联调完成后

全部4种传感器通过后：
1. 接上全部传感器
2. 烧debug固件
3. 拔掉O2传感器（避免UART4冲突）
4. USB-TTL接CN16看综合状态：T/H/CO2/HR/SpO2
5. 分别制造温度/CO2变化，观察继电器响应

## 故障速查表

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 串口无输出 | 未烧debug固件 / USB-TTL接错 | 确认-DHDICU_DEBUG / 检查CN16接线 |
| T=-99.9 | NTC断路或未接 | 量PA0电压 |
| T值异常大/小 | NTC B值或上拉电阻错 | 确认10kΩ@25°C B3950 |
| CO2=0 | 传感器未供电/TX-RX接反 | 逻辑分析仪抓PB11 |
| HR=0 SpO2=0 | 手指未贴紧/0x8A未发 | 确认红光透手指/抓PC12 TX |
| O2无帧 | 传感器未供电/TX-RX接反 | 逻辑分析仪抓PC11 |
| 蜂鸣器一直响 | COMM_FAULT(无屏幕板) | 正常设计行为 |
