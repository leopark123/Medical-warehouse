## 一、双工具协作规范

### 硬规则
- **代码只允许 Claude Code 改**
- **审计只允许 Codex 做**
- **最终接受只由人决定**
- **同一轮里，只允许一个工具改代码**

### 角色定义
| 角色 | 工具 | 职责 |
|------|------|------|
| 执行者 | Claude Code | 读代码、改代码、补测试、修编译、按审计意见修复 |
| 审计员 | Codex | diff审计、越界检查、回归风险、兼容性、测试覆盖、给结论 |
| 裁决者 | 人 | 编译烧录、实物验证、最终合并决策 |

### 标准作业流程（6步）
1. 人写 `CURRENT_TASK.md`（必须对应验收项编号）
2. Claude Code 实现
3. 人本地编译 + 烧录 + 基础验证
4. Codex 审计（输出 PASS / PASS WITH RISKS / BLOCK）
5. Claude Code 只修 BLOCK 和 Major 项
6. Codex 复审 → 人决定合并

### 四条纪律
1. Codex 审计期间不允许 Claude Code 改代码
2. Codex 不直接接管实现
3. 每轮任务只允许一个明确目标
4. **每轮任务必须对应验收项编号或开发阶段编号**

## 二、项目技术栈

### 主控板
- MCU: STM32F103VET6 (LQFP100, 72MHz, 512KB Flash, 64KB RAM)
- OS: FreeRTOS
- IDE: STM32CubeIDE
- 调试器: ST-Link V2 (SWD)
- 语言: C (C99)

### 屏幕板
- MCU: GD32F303RCT6 (与STM32F103RCT6 PIN TO PIN兼容，但有差异)
- OS: 裸机轮询（可选FreeRTOS）
- 库: GD32官方HAL库（推荐）或 STM32 HAL库逐项验证
- 注意: Flash页大小可能不同、时钟树差异(可达120MHz)、部分DMA通道映射不同

### 通信接口
| 串口 | 引脚 | 连接 | 波特率 | 备注 |
|------|------|------|--------|------|
| UART1 | PA9/PA10 | 屏幕板 | 115200 | 双板通信 |
| UART2 | PA2/PA3 | iPad | 115200 | 上位机通信 |
| UART3 | PB10/PB11 | CO2传感器 | 9600 | MWD1006E |
| UART4 | PC10/PC11 | O2传感器 | 9600 | OCS-3RL-2.0 |
| UART5 | PC12/PD2 | 体征模块 | 38400 | JFC103 |

## 三、主控板模块划分（FreeRTOS任务）

```
├── Core/                    # STM32CubeMX生成
├── Drivers/                 # HAL库驱动
├── Middlewares/FreeRTOS/    # FreeRTOS内核
├── App/
│   ├── Sensor/              # CO2/O2/NTC/体征 驱动与解析

│   ├── Alarm/               # 报警检测与处理
│   ├── Comm/                # 屏幕板通信 + iPad通信
```

| ControlTask | 高 | 200ms | 阈值+回差控制 + 继电器/PWM |
| AlarmTask | 高 | 100ms | 报警检测与处理 |

| SystemTask | 低 | 1s | 看门狗/时间统计/Flash保存 |

## 五、关键约束（互锁规则）

### 温度控制（回差±1°C）
- 待机: 设定值±1°C内 → 全OFF
- 制冷: >设定值+1°C → 压缩机+内风机
- 制热: <设定值-1°C → PTC加热+PTC风机
- **制冷与制热互斥**

### 湿度控制（回差±3%）
- 加湿: <设定值-3% → 加湿继电器ON
- 除湿: >设定值+3% → 联动制冷

### 氧浓度控制（回差-2%）
- 供氧: <设定值-2% → O2阀门ON

### 开放式供氧联动（安全关键）
| 功能 | 状态 | 原因 |
|------|------|------|
| O2阀门 | ON | 持续供氧 |
| 内循环 | **禁止** | 防氧气富集 |
| 外循环 | **强制开启** | 稀释通道 |
| 新风净化 | **强制开启** | 稀释通道 |
| 制冷 | **条件允许** | 仅在外循环/新风已开启时允许 |
| 制热 | **禁止** | 防燃烧 |
| 雾化 | **禁止** | 防耦合 |
| 紫外消毒 | **禁止** | 防紫外源 |

### 其他互斥
- 雾化 ↔ 紫外消毒：不能同时运行
- 紫外消毒 ↔ 开放式供氧：互相禁止
- 雾化期间：开放式供氧禁止启动

## 六、协议关键约束

### iPad协议 vs 双板协议
| 项目 | iPad协议 | 双板协议 |
|------|---------|---------|
| 帧头 | 0xAA | 0xAA 0x55 |
| 帧尾 | 0x55 | 0xED |
| 校验 | 含起始位: (AA+CMD+LEN+DATA)&0xFF | 不含帧头: (CMD+LEN+DATA)&0xFF |

### 参数传输规则
- 温度/湿度/氧浓度：协议值 = 实际值 × 10
- 屏幕板显示：湿度和氧浓度四舍五入取整
- 护理等级：1~3（不是1~4）
- iPad写入指令(0x03)：22字节全覆盖，主控不判断哪个改了

### Flash存储（已修正）
- 参数存储区：0x0807F000起（最后2扇区，4KB）
- 磨损均衡：双扇区轮换，版本号判断有效数据
- 运行时间：每10分钟保存一次

## 八、代码输出要求

1. **完整可编译** — 不允许伪代码、省略号、TODO占位
2. **遵循现有模块结构** — 新文件放到对应 App/ 子目录
3. **头文件独立** — 每个.c都有对应.h，接口清晰
4. **FreeRTOS规范** — 任务间通信用队列/信号量，不用全局变量裸访问
5. **不越界修改** — 只改 CURRENT_TASK.md 中声明的范围
6. **不顺手重构** — 修审计项时不扩大修改范围
7. **AFIO JTAG释放（强制）** — 所有含main()的文件必须在HAL_Init()之后立即加：
   ```c
   __HAL_RCC_AFIO_CLK_ENABLE();
   __HAL_AFIO_REMAP_SWJ_NOJTAG();
   ```
   原因：PB3(蜂鸣器)和PB4(WH继电器)是JTAG复用引脚，不释放则GPIO输出无效。
   此调用保留SWD(PA13/PA14)，不影响J-Link烧录调试。

## 九、禁止事项

- 禁止修改 Core/ 和 Drivers/ 中 CubeMX 自动生成的代码（除 USER CODE 区域）
- 禁止在中断处理函数中做阻塞操作
- 禁止在非UI任务直接操作显示（屏幕板职责）
- 禁止硬编码传感器校准值（应走参数配置）
- 禁止跳过校验和验证直接处理协议数据
- 禁止在开放式供氧期间允许制热/雾化/紫外（安全互锁）
