## 与冻结基线的差异说明

### 原理图.pdf — UART5网络名偏差
原理图第2页将UART5(PC12/PD2)标注为"串口5-wifi"，网络名为UART5-TX-WIFI/UART5-RX-WIFI。实际项目中该串口已改接JFC103体征模块(38400bps)。原理图的网络名未同步更新，属于硬件设计迭代遗留。**开发时以开发基线UART5=JFC103为准，不按原理图WiFi标注接线。**

### 功能表.pdf — 连接器细节 + 开放式供氧联动分歧
功能表包含更详细的硬件连接器编号（如CN9=红外灯、CN11=紫外灯等），这些细节在冻结总纲和开发基线中被抽象为信号名（如RED-IO、ZIY-IO）。如果开发中需要确认物理连接器对应关系，以功能表+原理图为准。

**开放式供氧联动逻辑分歧**：功能表第3页写"外循环打开，所有风扇、加热、制冷、除湿全部暂停"；冻结基线写"内循环禁止，外循环/新风强制开启，制热/雾化/紫外禁止，制冷条件允许"。功能表自身也注明该部分逻辑"后续程序设定再定"，说明功能表版本是初稿，冻结基线是最终定稿。**以冻结基线为准。**

### 检测指标说明.xlsx
JFC103模块本身具备更广泛的检测能力（HRV、心律失常分析等），但当前项目冻结范围明确只使用心率和血氧两项数据。体征数据响应(0x06)中字节6~13的血压/呼吸率字段固定填0。后续如需扩展，需重新评估并修订冻结总纲。

## 使用规则

1. **开发时遇到技术细节疑问** → 先查 `02_开发实现/PROJECT_RULES.md`
2. **PROJECT_RULES 未覆盖** → 查本目录对应规格书
3. **规格书与冻结基线有出入** → 以冻结基线为准
4. **需要硬件连接器级别细节** → 查功能表.pdf + 原理图.pdf
